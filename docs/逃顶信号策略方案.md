# 算力链逃顶信号系统 —— 策略方案

> 版本：v1.0（已落地）
> 研究框架：`research/OpenAI算力绑定_思维框架与投资指导.md`
> 制定日期：2026-07-19

## 1. 目标

综合三类信息源对 OpenAI 算力循环交易链做风险判级，符合风险预测时发出**逃顶信号**：

| 信息层 | 内容 | 获取方式 | 频率 |
|---|---|---|---|
| 价格层 | 链篮子情景 A/B/C、成员风险评分、传染性暴跌 | Tiingo 日线 | 每日自动 |
| 新闻层 | 链相关负面事件（IPO推迟/融资失败/营收放缓/capex削减等） | 新浪财经7×24快讯（默认）/ Tiingo News（可选，需付费权限） | 每日自动 |
| 仪表盘层 | 四指标：巨头capex环比、OpenAI下轮融资倍数、首个减速巨头、backlog欠款方集中度 | 前端手工录入 | 季度 |

信号输出三档：**绿（正常）｜黄（警惕）｜红（逃顶信号）**。
触达方式：前端页面 + 每日定时扫描存档 + 灯色变化主动推送（SMTP/webhook）。

## 2. 信号合成规则

### 2.1 各层评分

- **价格层**（0-100）：复用 `factor/openai_chain_sentinel.py` 的 `chain_regime()`：
  传染性暴跌计数、篮子 20/60 日动量、成员站上 MA60 占比、相对 QQQ 强度，评分 ≥60 判为情景 C。
- **新闻层加分**（0-20，`app/core/news_analyzer.py`）：近 7 日命中负面关键词的链相关新闻
  1-2 条 +5；3-4 条 +10；≥5 条 +15；命中重磅词（IPO推迟/融资失败级）额外 +5；
  命中当日链篮子跌 ≥3%（市场验证）再 +5；封顶 20。
  词表：`backend/data/sentinel_keywords.json`（重磅词/负面词/监控词三段，改词不改代码）。
- **仪表盘层加分**（0-30，`app/core/escape_signal.py`）：四项指标每项红灯 +7.5。

### 2.2 综合判级

综合评分 = 价格层 + 新闻层 + 仪表盘层，封顶 100。

- **红（逃顶信号）**，满足任一：
  1. 价格层评分 ≥60（情景 C，硬触发）；
  2. 综合评分 ≥70；
  3. 仪表盘 ≥2 项红灯 且 价格层评分 ≥40。
- **黄（警惕）**：综合评分 ≥40 或价格层判级 B。
- **绿（正常）**：其余。
- **推送去抖**：仅在灯色变化时推送与记录事件（`signal_events` 表）；首次扫描不推送。

## 3. 系统组成

### 3.1 后端

| 文件 | 职责 |
|---|---|
| `app/data/sina_news_client.py` | 新浪财经7×24快讯客户端（默认新闻源，免费无需 key） |
| `app/data/news_client.py` | Tiingo News 客户端（备选；免费 key 无 News API 权限会 403） |
| `app/core/news_analyzer.py` | 关键词匹配 + 新闻层评分 |
| `app/core/escape_signal.py` | 综合判级纯函数（全部规则集中于此，可单测） |
| `app/data/sentinel_store.py` | SQLite（`backend/data/sentinel.db`）：scan_daily / news_hits / dashboard_entries / signal_events |
| `app/core/chain_sentinel_service.py` | 编排：拉数→各层评分→综合判级→落库→（灯色变化时）推送 |
| `app/core/notify.py` | SMTP 邮件 + 通用 webhook 推送 |
| `app/api/chain_sentinel.py` | REST 接口 |
| `app/main.py` | lifespan 内挂 APScheduler 每日定时扫描 |
| `backend/scripts/run_sentinel_scan.py` | 独立扫描脚本（Windows 计划任务兜底） |

### 3.2 REST 接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/chain-sentinel/status` | 最新综合状态（灯色/评分/各层明细/仪表盘灯态） |
| POST | `/api/chain-sentinel/scan` | 立即扫描（按交易日幂等覆盖；灯色变化时推送） |
| GET | `/api/chain-sentinel/history?days=90` | 每日判级历史 |
| GET | `/api/chain-sentinel/news?days=7` | 近期命中新闻 |
| GET/POST | `/api/chain-sentinel/dashboard` | 仪表盘四指标录入查询/更新 |
| GET | `/api/chain-sentinel/events` | 信号事件（灯色变化与推送记录） |

### 3.3 前端

`frontend/src/pages/ChainSentinel/`：灯色卡片（绿/黄/红 + 综合评分 + 判级理由 + 立即扫描）、
价格层成员明细、新闻命中列表、仪表盘四指标录入表单、历史判级曲线与事件列表。
路由 `/chain-sentinel`，菜单「链风险哨兵」。

### 3.4 定时与推送配置

`backend/data/sentinel_config.json`：

- `news.provider`：新闻源，`sina`（默认，新浪财经7×24，免费无需 key）或 `tiingo`（需付费权限）。
- `scheduler`：默认每日 05:30（Asia/Shanghai，美股收盘后）扫描；冬令时改 6:30。
- `smtp`：QQ/163 邮箱授权码；`enabled=true` 且字段齐全才启用。
- `webhook`：POST JSON `{title, text}`，兼容 Server酱/Bark/企业微信机器人。
- 两通道均未启用时：只存档，不推送。

新闻关键词词表：`backend/data/sentinel_keywords.json`，中英双语（重磅词/负面词/监控词），改词不改代码。

## 5. 使用方式

1. 日常：系统自动在每日 05:30 扫描并存档；灯色变化时按配置推送。
2. 手动：前端「链风险哨兵」页点「立即扫描」，或 `POST /api/chain-sentinel/scan`，
   或直接跑 `backend/scripts/run_sentinel_scan.py`。
3. 每季度：在页面录入四指标灯态（capex 环比、OpenAI 下轮倍数、首个减速巨头、backlog 集中度）。
4. 收到红色信号后的动作对应研究文档落地纪律：循环破裂 → 降 β、现金/对冲。

## 6. 配额与边界

- Tiingo 免费档有小时/日请求限额：每次扫描约 9 次价格请求，每日一次足够；
  手动频繁触发可能撞 429（等配额恢复后重试即可）。
- 新闻默认走新浪财经7×24（免费、无配额顾虑）；Tiingo News 需付费权限，免费 key 会 403。
- 新闻分析为规则词表，不引入 LLM；仪表盘三项指标人工录入，不做财报自动抓取。
- 不接实盘、不做自动交易；信号用于提醒与回测过滤。
