# 行情多数据源与回测平台选择方案

## 1. 现状问题

当前系统仅接入 **Gate.io**，存在以下限制：

| 问题 | 现状 | 影响 |
|---|---|---|
| 历史 K 线长度受限 | Gate.io 最多返回最近 **10,000 根 K 线** | 前端默认只能选近 3 个月；1h 级别最多约 1.1 年 |
| 数据源单一 | 仅 Gate.io 一个行情源 | 数据质量、币种覆盖、连续性无法横向对比 |
| 回测不能选平台 | `BacktestRequest` 无 `provider` 字段 | 无法验证同一策略在不同交易所数据的稳健性 |
| 币种支持有限 | 依赖 Gate.io 上架币种 | 主流币足够，但小币种、链上概念币覆盖不足 |

## 2. 目标

1. **更长的历史数据**：至少支持 3~5 年、1h 级别 K 线。
2. **更多币种**：覆盖主流现货 + 合约交易对，并能方便扩展。
3. **回测时可选择行情平台**：同一 symbol/周期可切换不同交易所对比。
4. **向后兼容**：保留 Gate.io，作为默认数据源。

## 3. 可选行情 API 对比

以下均为 **免费公有行情接口**（无需付费订阅即可获取 K 线），按历史长度、币种数量、稳定性排序：

| 交易所 | 公有 K 线接口 | 历史长度 | 1h K 线最大回推 | 币种数量 | 速率限制 | 特点 |
|---|---|---|---|---|---|---|
| **Binance** | `/api/v3/klines` (现货)<br>`/fapi/v1/klines` (U 本位合约) | 现货约 2017 年起<br>合约约 2019 年起 | 约 **8 年** | 2000+ | 1200 req/min/IP | 历史最久、币种最多、文档最完善 |
| **OKX** | `/api/v5/market/history-candles` | 约 2019 年起 | 约 **6 年** | 500+ | 20 req/s/IP | 历史长、合约数据全、symbol 格式统一 |
| **Bybit** | `/v5/market/kline` | 现货约 2020 年起<br>合约约 2018 年起 | 约 **6 年** | 500+ | 50 req/s/IP | 合约深度好、API v5 统一 |
| **Gate.io** | `/spot/candlesticks`<br>`/futures/{settle}/candlesticks` | 最近 10,000 根 | 约 **1.1 年** (1h) | 2000+ | 200 req/min/IP 约 | 当前已接入 |

> 注：以上数值为官方公开文档与实测经验，实际可能随业务调整。Binance 对 1 个 IP 的访问在某些地区可能受限，可考虑 IP 白名单或轻代理。

### 结论

- **首选接入 Binance**：历史最久、币种最多、公有接口最稳定，性价比最高。
- **次选 OKX / Bybit**：作为补充，特别是合约数据与 Binance 做交叉验证。
- **保留 Gate.io**：作为默认 fallback，避免破坏现有逻辑。

## 4. 推荐架构

采用 **适配器模式 + 统一数据客户端**：

```
backend/app/data/
├── __init__.py
├── base_client.py          # 抽象接口：fetch_klines / get_spot_symbols / get_futures_symbols
├── binance_client.py       # Binance 适配器
├── okx_client.py           # OKX 适配器（可选）
├── bybit_client.py         # Bybit 适配器（可选）
├── gateio_client.py        # 现有，改造为适配器
├── client_factory.py       # 根据 provider 返回对应 client
├── cache_manager.py        # 增加 provider 维度
└── data_store.py           # 增加 provider 维度
```

### 4.1 统一数据模型

无论哪个交易所，统一归一化字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `timestamp` | datetime[ns, UTC] | K 线开始时间 |
| `open` | float | 开盘价 |
| `high` | float | 最高价 |
| `low` | float | 最低价 |
| `close` | float | 收盘价 |
| `volume` | float | 成交量 |
| `quote_volume` | float | 成交额（可选） |
| `symbol` | str | 归一化交易对，如 `BTC_USDT` |
| `interval` | str | `1m/5m/15m/30m/1h/4h/1d` |
| `market_type` | str | `spot/futures_usdt` |
| `provider` | str | `binance/okx/bybit/gateio` |

### 4.2 缓存键增加 provider

原缓存键：`{symbol}_{interval}_{market_type}`  
新缓存键：`{provider}_{symbol}_{interval}_{market_type}`  
存储目录也按 provider 分：`data/cache/{provider}/...`

这样不同交易所的同 symbol 数据互不覆盖，回测可切换数据源。

#### `backend/app/models/enums.py`

```python
class Provider(str, Enum):
    GATEIO = "gateio"
    BINANCE = "binance"
    OKX = "okx"
    BYBIT = "bybit"

# 同时扩展 Interval，开放更多周期
class Interval(str, Enum):
    ONE_MINUTE = "1m"
    FIVE_MINUTES = "5m"
    FIFTEEN_MINUTES = "15m"
    THIRTY_MINUTES = "30m"
    ONE_HOUR = "1h"
    FOUR_HOURS = "4h"
    EIGHT_HOURS = "8h"
    ONE_DAY = "1d"
    ONE_WEEK = "7d"
```

#### `backend/app/models/schemas.py`

在 `KlineRequest`、`FactorEvalRequest`、`BacktestRequest` 中增加 `provider: Provider = Provider.GATEIO`。

> 注意：为保持旧回测结果可读取，数据请求字段加默认值；存储结果中同步记录 `provider`。

### 5.2 抽象数据客户端

#### 新建 `backend/app/data/base_client.py`

```python
from abc import ABC, abstractmethod
from datetime import datetime
from typing import Any

import pandas as pd

from app.models.enums import Interval, MarketType


class MarketDataClient(ABC):
    """行情数据源抽象基类"""

    @property
    @abstractmethod
    def name(self) -> str:
        """provider 名称，如 binance"""

    @abstractmethod
    async def fetch_klines(
        self,
        symbol: str,
        interval: Interval,
        market_type: MarketType,
        start_time: datetime,
        end_time: datetime,
    ) -> pd.DataFrame:
        """拉取 K 线，返回统一 OHLCV DataFrame"""

    @abstractmethod
    async def get_spot_symbols(self) -> list[dict[str, Any]]:
        """返回统一格式 SymbolInfo 列表"""

    @abstractmethod
    async def get_futures_symbols(self, settle: str = "usdt") -> list[dict[str, Any]]:
        """返回统一格式 SymbolInfo 列表"""

    async def close(self):
        """释放连接，子类按需实现"""
        pass
```

### 5.3 改造 `gateio_client.py`

让 `GateIOClient` 继承 `MarketDataClient`，并增加 `name = "gateio"`。

### 5.4 新增 `binance_client.py`

实现要点：

- Base URL：`https://api.binance.com`（现货）、`https://fapi.binance.com`（U 本位合约）
- K 线接口：
  - 现货：`GET /api/v3/klines`
  - 合约：`GET /fapi/v1/klines`
- 参数：`symbol=BTCUSDT&interval=1h&startTime=...&endTime=...&limit=1000`
- Symbol 映射：
  - 系统内统一格式 `BTC_USDT`，Binance 格式 `BTCUSDT`，需在适配层互转。
  - 合约统一视为 USDT 本位，暂时不接入币本位。
- 分页：Binance 单次 limit 最大 1000，按 `last_open_time + interval_ms` 继续请求，直到满足 end_time。
- 字段映射：Binance 返回 `[open_time, open, high, low, close, volume, close_time, quote_volume, ...]`，取前 6 列后重命名。

### 5.5 新增 `client_factory.py`

```python
from app.data.binance_client import BinanceClient
from app.data.gateio_client import GateIOClient
from app.data.base_client import MarketDataClient
from app.models.enums import Provider


_CLIENTS: dict[Provider, type[MarketDataClient]] = {
    Provider.GATEIO: GateIOClient,
    Provider.BINANCE: BinanceClient,
}


def get_client(provider: Provider) -> MarketDataClient:
    client_cls = _CLIENTS.get(provider)
    if not client_cls:
        raise ValueError(f"不支持的行情源: {provider}")
    return client_cls()
```

### 5.10 配置与 API Key

当前 Gate.io 使用公有接口，无需 Key。**Binance/OKX/Bybit 的 K 线接口同样为公有接口，也无需 Key**。因此改造第一阶段无需引入密钥管理。

若后续需要：
- 更高频率 / 更低延迟的 WebSocket 行情
- 私有接口（账户、下单）

再引入环境变量：`BINANCE_API_KEY`、`BINANCE_API_SECRET` 等。

## 6. 实施优先级建议

| 阶段 | 内容 | 预计工时 | 收益 |
|---|---|---|---|
| **P0** | 接入 Binance 现货 + U 本位合约；后端统一客户端；缓存加 provider | 1~2 天 | 历史数据从 1 年扩展到 5~8 年；币种大幅增加 |
| **P1** | 前端增加 provider 选择器；交易对按 provider 拉取；回测结果记录 provider | 0.5~1 天 | 用户可在回测时切换平台 |
| **P2** | 接入 OKX / Bybit 作为补充 | 1~2 天 | 多源交叉验证，提升稳健性 |
| **P3** | 扩展 Interval 枚举到 1m/5m/15m/30m 等；前端开放更多周期 | 0.5 天 | 支持中高频策略回测 |
| **P4** | 引入 API Key 管理；支持 WebSocket 实时行情 | 2~3 天 | 为实盘/模拟盘做准备 |

## 7. 风险提示

1. **地区访问限制**：Binance API 在某些地区可能无法直接访问，需测试网络连通性。
2. **Rate Limit**：公有接口有 IP 限速，长周期全量补数据时需控制请求频率，避免被封。
3. **Symbol 命名差异**：不同交易所的 symbol 格式不同，必须统一在适配层转换。
4. **数据一致性**：不同交易所的 K 线可能在极端行情下存在微小差异，回测结果会略有不同。
5. **币种上线时间**：即使是 Binance，也不是所有币种都从 2017 年开始，某些小币种历史可能只有几个月。

## 8. 预期效果

完成 P0 + P1 后：

- 1h 级别回测可倒推 **5 年以上**，4h/1d 级别可覆盖更久。

- 回测页面新增“行情平台”下拉框，可在 Gate.io / Binance 间切换。
- 旧回测结果与默认行为保持兼容。
